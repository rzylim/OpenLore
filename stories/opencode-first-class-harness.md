# Make OpenCode a First-Class Harness in OpenLore

## Goal

Make opencode a first-class harness in OpenLore by giving it the same integration depth Claude Code has — auto-detection, an install adapter, auto-orient on session start, and clean install/uninstall — adapted for OpenCode's TypeScript plugin model.

## Acceptance Criteria

- [x] `openlore install --agent opencode` creates `.opencode/plugins/agent-guard.ts` containing the expected launcher shim
- [x] Re-running `openlore install --agent opencode` is a no-op (idempotent — file content unchanged)
- [x] `openlore install --uninstall --agent opencode` removes `.opencode/plugins/agent-guard.ts`
- [x] `openlore install` without `--agent` auto-detects OpenCode when `.opencode/opencode.json` or `~/.config/opencode/opencode.json` exists
- [x] Installed `agent-guard.ts` contains a `npx --yes openlore orient --json` spawn in its `experimental.chat.system.transform` hook
- [x] `openlore setup --tools opencode` continues to install 8 skill files to `.opencode/skills/` (regression)
- [x] Pre-existing non-OpenLore files in `.opencode/plugins/` survive uninstall

## Won't Do

- No new OpenSpec domain for opencode (infrastructure, not domain logic)
- No changes to `omoa` target or its SDD plugins (enforcer, decision-extractor, context-injector, anti-laziness)
- No plugins compiled to JS — raw `.ts` is sufficient (OpenCode natively loads them)
- No plugin version manager or upgrade automation
